# 启动模式设计（当前实现）

## 模式选择方式

开机时检测 **SA 按钮**（GPIO36）：
- **SA 短按（<3s）或无操作** → 进入默认模式（当前为 WiFi AP）
- **SA 长按 3s** → 进入摇杆选择菜单，通过右摇杆方向选择模式（超时 5s 回退默认）

### 右摇杆方向映射

| 方向 | 条件 | 启动模式 |
|------|------|----------|
| ⬆️ 上 | pitch < 1300 | **USB Xbox** 360 手柄 |
| ⬇️ 下 | pitch > 1700 | **BLE FPV** HID 手柄 |
| ➡️ 右 | roll > 1700 | **USB U盘 (MSC)** — TF 卡直读 |
| ⬅️ 左 | roll < 1300 | **透穿调参** — BLE NUS ↔ USB CDC |
| SD 按钮 | GPIO38 == 0 | **WiFi AP** — Web 配置页面 |
| 超时 5s | 未选择 | 回退默认模式 |

## 默认模式

当前默认模式为 **WiFi AP**。上电后自动开启 WiFi 热点 + WebSocket 配置页，可通过 Web UI 配置校准、CRSF、LED 等参数。

## 模块初始化策略

| 功能模块 | 启用条件 |
|----------|----------|
| ADC (摇杆) | 始终启用（模式选择依赖摇杆值） |
| LED | 始终启用 |
| 音频 | 始终启用（模式播报、告警） |
| CRSF 射频 | RF / BLE / Passthrough / WiFi 模式 |
| BLE HID | BLE 模式 |
| BLE NUS | Passthrough 模式 |
| USB HID | USB Xbox 模式 |
| USB MSC | U盘模式（独占，不进主循环） |
| USB Host (CDC) | Passthrough 模式 |
| WiFi AP | WiFi 模式 |

## 按键引脚

| 引脚 | 功能 |
|------|------|
| SA (GPIO36) | 模式选择修饰键（长按 3s 进选择菜单） |
| SB (GPIO37) | 2 段开关（解锁/锁定） |
| SC (GPIO38~40) | 3 段开关 |
| SD (GPIO38) | 模式选择快捷键（WiFi/U盘连接） |

## 自动对频

当 CRSF 已初始化但链路未连接时，自动对频状态机会定期发送 bind 命令，直到接收机上线。

## 实现位置

- 模式枚举：`main.c` → `boot_mode_t`
- 模式检测：`main.c` → `detect_boot_mode()`
- 默认模式：`main.c` → `get_default_mode()`
- 模式调度：`main.c` → `app_main()`
