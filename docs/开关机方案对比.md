# 遥控器开关机方案对比

## 方案一：深睡眠（推荐，Xbox 方案）

| 项目 | 说明 |
|------|------|
| 原理 | ESP32 深睡眠 + RTC GPIO 唤醒 |
| 硬件 | 按键 → RTC GPIO + GND，**无需额外元件** |
| 关机操作 | 长按 → 保存 NVS → `esp_deep_sleep_start()` |
| 唤醒操作 | 短按 → RTC GPIO 触发重启 |
| 静态功耗 | ~10μA |
| 可保存数据 | 关机前 `do_nvs_save()` 写入校准和配置 |

**支持 RTC 唤醒的 GPIO：** 0, 2, 4, 12-15, 25-27, 32-39

```c
// 配置唤醒
esp_sleep_enable_ext0_wakeup(GPIO_NUM_33, 0);  // 下拉唤醒

// 关机
void power_off(void) {
    request_nvs_save();           // 确保配置已保存
    vTaskDelay(pdMS_TO_TICKS(100));
    esp_deep_sleep_start();       // 不会再回来
}
```

**代表产品：** Xbox 手柄（含 Xbox 360/One/Series X）。

**优点：** 零硬件成本、代码简单、功耗够低。  
**缺点：** 关机后完全死透，不能做定时关机。

## 方案二：软开关 + 自锁电路（RadioMaster 方案）

| 项目 | 说明 |
|------|------|
| 原理 | P-MOS 自锁 + MCU GPIO 维持导通 |
| 硬件 | PMOS + 按键 + 电阻（~5 元件） |
| 关机操作 | 长按 → GPIO 拉低 → MOS 关断 → 整机断电 |
| 唤醒操作 | 按键按下 → MOS 导通 → MCU 得电启动 |
| 静态功耗 | 0（完全断电） |
| 可保存数据 | GPIO 释放前执行 NVS 保存 |

**电路示意：**

```
3.7V ────┬─── PMOS ──── 3.3V LDO ──── ESP32
         │     │
         └─────┘
        按键      GPIO
```

开机：按按键 → PMOS 导通 → ESP32 启动 → GPIO 拉高维持导通  
关机：长按 → GPIO 释放 → PMOS 关断 → 全机断电

**代表产品：** RadioMaster TX16S / Zorro / Boxer。

**优点：** 零待机电流、不挑 GPIO、可实现物理级断电。  
**缺点：** 需要额外元件（PMOS 和电阻）、PCB 飞线。

---

## 方案三：纯硬件物理开关（玩具级）

| 项目 | 说明 |
|------|------|
| 原理 | 拨动开关直接切断电池 |
| 硬件 | 单刀双掷开关 × 1 |
| 操作 | 拨动即断电/上电 |
| 静态功耗 | 0 |
| 可保存数据 | **不能**，瞬间断电 |

**代表产品：** 百元级玩具控、早期天地飞。

**优点：** 最简单、零代码、零成本。  
**缺点：** 关机不能保存、不能防误触、开关寿命有限。

---

## 对比总结

| | 深睡眠 | 软开关自锁 | 物理开关 |
|--|--------|-----------|---------|
| 额外硬件 | 无 | PMOS + 电阻 | 拨动开关 |
| 待机功耗 | ~10μA | 0 | 0 |
| NVS 保存 | ✅ 可保存 | ✅ 可保存 | ❌ |
| 长按关机 | ✅ | ✅ | ❌ |
| 自动低电关机 | ✅ | ✅ | ❌ |
| 代码复杂度 | 低 | 中 | 无 |
| 代表产品 | Xbox 手柄 | RadioMaster | 玩具控 |

---

## 推荐：深睡眠

你的 ESP32 已经有 ADC（20kHz 连续采样）、CRSF、WiFi AP，不需要再增加硬件复杂度。一个按键接到 RTC GPIO，按→唤醒/长按→关机，跟 Xbox 手柄一样。
